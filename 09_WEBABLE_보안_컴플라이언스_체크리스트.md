# WEBABLE 보안/컴플라이언스 체크리스트

## 1. 멀티테넌트 격리

- [ ] 모든 핵심 테이블에 `org_id` 또는 `site_id`가 포함되어 있다.
- [ ] PostgreSQL Row Level Security 정책이 적용되어 있다.
- [ ] API 계층과 DB 계층 모두에서 테넌트 scope를 검증한다.
- [ ] 관리자 권한 우회 테스트가 CI에 포함되어 있다.
- [ ] AI Tool Runtime이 사용자 권한을 초과해 데이터를 조회/수정할 수 없다.

## 2. 인증/권한

- [ ] Owner/Admin/Designer/Marketer/CS/Fulfillment/Viewer 기본 역할이 정의되어 있다.
- [ ] 결제, 삭제, 개인정보 export, AI 데이터 사용 변경은 Owner 또는 명시 권한만 가능하다.
- [ ] MFA, 세션 만료, 비밀번호 정책, 계정 잠금 정책이 정의되어 있다.
- [ ] 감사 로그에 actor, action, target, before/after, ip, userAgent가 기록된다.

## 3. 개인정보

- [ ] 수집 항목, 이용 목적, 보존기간, 파기 절차가 사이트별 개인정보처리방침에 반영된다.
- [ ] 회원/주문/예약/문의 데이터의 export/delete 요청 프로세스가 있다.
- [ ] 위탁 처리 업체(PG, 문자, 이메일, 클라우드, AI provider 등)가 관리된다.
- [ ] 개인정보가 포함된 로그/프롬프트/첨부파일의 보존기간이 제한된다.
- [ ] 운영자 화면에서 불필요한 개인정보 원문 노출을 줄인다.

## 4. AI 프라이버시/안전성

- [ ] 기본 정책은 고객 데이터를 모델 학습에 사용하지 않는 것이다.
- [ ] 학습 활용이 필요한 기능은 opt-in과 opt-out 절차를 제공한다.
- [ ] AI job마다 프롬프트, 데이터 범위, 모델, 비용, 결과, 승인자를 기록한다.
- [ ] 주문/회원/예약 데이터는 AI 처리 전 최소화 또는 마스킹한다.
- [ ] AI 생성 결과는 초안으로 표시하고 운영자 승인 후 적용한다.
- [ ] 고위험 표현(의료, 금융, 건강 효능, 법률 등)에 경고/검수 플래그를 제공한다.
- [ ] AI 코드 생성은 샌드박스, 정적 분석, CSP, XSS/SSRF 차단을 통과해야 한다.

## 5. 전자상거래

- [ ] 사업자 정보, 통신판매업 정보, 고객센터 정보가 사이트 하단/정책 페이지에 표시된다.
- [ ] 상품 가격, 배송비, 환불/교환/반품 조건이 명확히 표시된다.
- [ ] 주문/결제/취소/환불/배송 상태 변경 이력이 보존된다.
- [ ] PG 승인/취소/환불 웹훅은 서명 검증과 idempotency를 적용한다.
- [ ] 부분 취소, 부분 배송, 추가 결제, 배송비 변경을 상태 머신과 원장으로 처리한다.

## 6. 인프라/운영

- [ ] WAF, Rate Limit, Bot Protection 기본 정책이 있다.
- [ ] 비밀키는 KMS/Secret Manager로 관리하고 코드에 저장하지 않는다.
- [ ] 정기 백업, 복구 리허설, RPO/RTO가 정의되어 있다.
- [ ] 장애 대응 Runbook, 온콜, 상태 페이지, 고객 공지 프로세스가 있다.
- [ ] 로그, 메트릭, 트레이스, AI 비용 모니터링이 구축되어 있다.

## 7. 출시 전 필수 승인

- [ ] 법무: 약관, 개인정보처리방침, AI 이용정책, 환불정책 검토 완료
- [ ] 보안: 취약점 진단, 권한 테스트, 백업 복구 리허설 완료
- [ ] 제품: MVP P0 기능 acceptance criteria 통과
- [ ] 운영: CS 매뉴얼, 장애 대응, 고객 온보딩 문서 완료
